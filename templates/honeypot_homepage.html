01 — Introduction

What is HoneyPot IDS?

// System Overview

The Honeypot-Based Intrusion Tracking System is a fake system engineered to attract attackers and silently monitor their activities. It records attacker details including IP address, access time, and login attempts — then displays structured alerts on a web dashboard for administrators.

Built using Flask, HTML/CSS, SQLAlchemy and deployed on a virtual Linux environment, it serves as a powerful decoy and forensic data collection tool in cybersecurity.

// Problem to be Solved

  • Networks constantly targeted using password guessing, port scanning & malicious requests.
  • Traditional tools focus on prevention — not on analyzing attacker behavior.
  • Administrators lack detailed information about intruders post-attack.
  • Difficult to understand attacker methodologies and improve defenses proactively.
  • Zero-day and unknown attack patterns go undetected by conventional IDS.
02 — Architecture

System Modules

Seven specialized components working in concert to detect, capture, analyze, and alert on malicious network activity.

M-01⬡

Honeypot Service

Creates fake network services (SSH/HTTP) to attract attackers. Mimics a real system to safely observe intrusion behavior without risk.

M-02⊕

Request Capture

Captures all incoming attacker requests including IP address, port, timestamp, and raw input data for monitoring and analysis.

M-03◎

Intrusion Logging

Records every intrusion attempt in the database using SQLAlchemy ORM, stored securely for forensic investigation.

M-04⧖

Intrusion Analysis

Analyzes logged data to detect suspicious patterns — repeated IPs, abnormal behavior, and brute-force signatures.

M-05⚡

Alert Generation

Generates real-time alerts on suspicious activity, enabling administrators to take immediate defensive action.

M-06▣

Dashboard Module

Displays intrusion statistics, recent attacks, and alerts in a structured web interface for easy security monitoring.

M-07⬛

Database Management

Manages all database operations — inserting, retrieving, and deleting intrusion records — using SQLAlchemy ORM for efficient and reliable data handling.
